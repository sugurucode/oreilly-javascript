## 予想

"A" と "B" が出力され続け、"Hello, world!" は永久に出力されない。

## 結果

↑であってた。
きれいに交互に表示

## 理由

JavaScriptのイベントループは、「タスク（マクロタスク）」と「マイクロタスク」という2種類のキューを管理しており、実行優先度が異なる。

1.  **タスクの登録:**
    `setTimeout` は、「**タスク**」をスケジュール（"setTimeout()...対応するコールバックがタスクキューに追加されます"）。
    なので "Hello, world!" はタスクキューに入る。

2.  **マイクロタスクの登録:**
    `async function` 内の `await` は `Promise` に基づいており、これは「**マイクロタスク**」をスケジュール（"JavaScript のプロミス...どちらもコールバック実行にマイクロタスクキューを使用します"）。
    マイクロタスクキューに入る

3.  **実行の優先順位:**
    「イベントループの一回の処理中に、タスクキューで最も古い実行可能なタスクが実行されます。**その後、マイクロタスクキューが空になるまでマイクロタスクが実行され**...」

4.  **無限ループの問題:**
    `longA` と `longB` は、実行される（マイクロタスクが処理される）たびに、`while(true)` ループと `await` によって**即座に次のマイクロタスクをキューに追加します**。

setTimeout が呼ばれ、「1秒後に "Hello, world!" を実行する」という未来のタスクがスケジュール。
longA() と longB() が呼び出され、即座にマイクロタスクがキューに追加される。そして、この「最初のタスク（スクリプト全体の実行）」が終わった直後に、ルール通り「マイクロタスクキューが空になるまで」の処理が始まり、無限ループが発生
