1. defer は「準備ができたら実行する」機能だから安全

読み込み順序を守る: index1.js は jQuery などを使っているため、ライブラリより後に実行されないとエラーになります。defer は「書かれた順番通り」に実行してくれるので安心です（async だと順番がバラバラになり、エラーになります）。

画面の要素待ちをしてくれる: JS が実行されるのは「HTML がすべて読み込まれた後」です。そのため、画面の下の方にある <div id="1000"> も確実に見つかります（async だと見つかる前に実行してしまい、エラーになることがあります）。

2. defer は「読み込み待ち」をしないから速い

普通の <script> タグは、ファイルをダウンロードして実行するまで HTML の表示（解析）を止めてしまいます。

defer をつけると、裏側でファイルをダウンロードしながら HTML の表示を進めてくれるため、ページが表示されるまでの時間が短縮されます。
